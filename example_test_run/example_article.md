# Article Style Guide

## Voice and Tone

- Write in second person ("you") when addressing the reader directly
- Use active voice over passive voice
- Keep sentences short — aim for under 20 words
- Explain jargon on first use, then use the short form

## Structure

Every article must include:
1. A hook in the first paragraph (a surprising fact, a question, or a bold claim)
2. A brief "what you'll learn" section
3. Code examples for any technical concept
4. A concrete takeaway or next step at the end

## Formatting

- Use `##` for section headers, `###` for sub-sections
- Wrap all code in fenced code blocks with a language tag
- Use bullet lists for 3+ items; write prose for fewer than 3
- Bold (**text**) for key terms on first use; do not overuse

## Length

- Aim for 600–900 words
- If the topic demands more depth, split into a series
